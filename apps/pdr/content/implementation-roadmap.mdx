## 6.1 Phase 0: Proof of Concept (Current State)

A testing repository exists today at [github.com/fiannello-por/data-analytics-demo](https://github.com/fiannello-por/data-analytics-demo). It demonstrates the core concepts:

- Lightdash charts and dashboards defined as YAML in Git
- CI/CD pipeline with automated validation on every push
- AI-powered PR review generating structured feedback _(optional — uses OpenAI API)_
- Automated changelog generation from merged pull requests _(optional — uses OpenAI API)_
- Semantic layer with metric definitions

<Callout type="warn">
  **This is a POC — not a production system.** It's available for the team to
  explore, test, and evaluate. It is subject to modifications at any time and
  should be treated as a sandbox for proving out the architecture.
</Callout>

## 6.3 Phase 2: dbt Integration

Replace BigQuery scheduled queries with version-controlled dbt models. This phase is independent of Phase 1 — the dashboards and semantic layer built in Phase 1 carry forward without rework.

**Deliverables:**

- dbt project initialized on top of existing BigQuery tables
- Scheduled queries migrated to dbt models (prioritized by business domain)
- Built-in data tests and documentation
- Lineage tracking (which dashboards are affected by upstream changes)

**Prerequisites:**

- Phase 1 stable and in production use
- Team comfortable with the PR-based workflow

## 6.4 Parallel Run Guarantee

At every phase, the existing Looker Studio dashboards remain live and untouched. There is no "switch-over day."

```text
Phase 0          Phase 1                    Phase 2
─────────────────────────────────────────────────────────▶

Looker Studio ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━▶  (stays live)

                  Lightdash ━━━━━━━━━━━━━━━━━━━━━━━━━━━━▶  (grows alongside)

                                             dbt ━━━━━━━▶  (plugs in when ready)
```

- Looker Studio stays live until the team is confident in the replacement
- Each phase delivers standalone value — if the project pauses at any point, what's been built remains useful
- The team decides when (and whether) to deprecate Looker Studio dashboards
