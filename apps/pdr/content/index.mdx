## Executive Summary

The analytics team has been asked to evaluate replacing Looker Studio with a platform that delivers full flexibility, role-level security, version control, and AI-iterability — while eliminating platform lock-in.

This document proposes an architecture built entirely on **open-source tools** managed through GitHub. Instead of building a custom web application from scratch, we leverage **[Lightdash](https://docs.lightdash.com/)** — an open-source BI platform purpose-built for code-defined dashboards, semantic layers, and team collaboration — on top of our existing BigQuery warehouse.

## What This Delivers

| Requirement                         | How it's addressed                                                                 |
| ----------------------------------- | ---------------------------------------------------------------------------------- |
| **Code-defined dashboards**         | Every chart and metric is a YAML file in Git — reviewable, testable, deployable    |
| **Version control with changelogs** | Full Git history on every change, with automated changelog generation              |
| **Role-level security**             | Folder-based access control with personal and shared production spaces             |
| **AI-iterability**                  | Claude Code reads, writes, and modifies dashboards because they're just text files |
| **No platform lock-in**             | Open-source stack, everything exportable, no proprietary formats                   |

## MVP Target

The **Sales Performance Dashboard** is the first dashboard to be rebuilt. The architecture is designed to scale from this single dashboard to the full reporting suite, with a clear path to add a dbt transformation layer over time.
