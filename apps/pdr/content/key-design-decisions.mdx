## Decision Framework

Each decision below was evaluated against four criteria: **cost**, **time-to-value**, **team capability**, and **long-term maintainability**.

## D1: Data Warehouse Platform

<Tabs items={['Snowflake (Selected)', 'BigQuery', 'Redshift']}>
  <Tab value="Snowflake (Selected)">
    **Score: 9.2/10**

    - Near-zero admin overhead with auto-scaling
    - Native dbt integration with incremental models
    - Separate compute/storage billing model
    - Strong ecosystem of partner integrations

    <Callout type="info">
    **Recommendation**: Snowflake is the clear winner given our existing dbt investment and the team's familiarity with the platform.
    </Callout>
  </Tab>
  <Tab value="BigQuery">
    **Score: 7.8/10**

    - Excellent for ad-hoc queries with slot-based pricing
    - Strong ML integration with BigQuery ML
    - Less mature dbt adapter compared to Snowflake
    - Team has no existing GCP expertise
  </Tab>
  <Tab value="Redshift">
    **Score: 6.5/10**

    - Deep AWS integration (existing cloud provider)
    - Requires more manual tuning and maintenance
    - Fixed cluster sizing leads to over-provisioning
    - dbt adapter is stable but less feature-rich
  </Tab>
</Tabs>

## D2: BI / Visualization Tool

<Tabs items={['Lightdash (Selected)', 'Looker', 'Tableau']}>
  <Tab value="Lightdash (Selected)">
    **Score: 8.7/10**

    - Native dbt integration (metrics layer as code)
    - Open source with self-hosted option
    - Git-based workflow matches engineering practices
    - $0 license cost (self-hosted)

    <Callout type="info">
    **Recommendation**: Lightdash aligns with our dbt-first strategy and eliminates BI license costs entirely.
    </Callout>
  </Tab>
  <Tab value="Looker">
    **Score: 7.5/10**

    - Powerful semantic modeling (LookML)
    - Requires separate modeling layer (duplicates dbt)
    - $125K/year minimum license
    - Steep learning curve for business users
  </Tab>
  <Tab value="Tableau">
    **Score: 6.0/10**

    - Best-in-class visualization capabilities
    - No native dbt integration
    - $180K/year for 200 users
    - Extract-based model introduces data staleness
  </Tab>
</Tabs>

## D3: Orchestration Strategy

<Tabs items={['GitHub Actions (Selected)', 'Airflow', 'Dagster']}>
  <Tab value="GitHub Actions (Selected)">
    **Score: 8.0/10**

    - Already in use for CI/CD — no new tool to manage
    - Native integration with dbt Cloud CLI
    - Sufficient for our current pipeline complexity
    - Zero additional infrastructure cost

    <Callout type="info">
    **Recommendation**: GitHub Actions is "good enough" for our current scale. We can revisit if pipeline complexity increases significantly in Phase 3.
    </Callout>
  </Tab>
  <Tab value="Airflow">
    **Score: 7.0/10**

    - Industry standard for complex DAGs
    - Significant operational overhead (self-managed)
    - Overkill for our current 15-model pipeline
    - $30K/year for managed (Astronomer/MWAA)
  </Tab>
  <Tab value="Dagster">
    **Score: 7.5/10**

    - Modern, asset-based orchestration
    - Excellent dbt integration
    - Team would need to learn a new paradigm
    - $24K/year for Dagster Cloud
  </Tab>
</Tabs>
